# Simple To-Do List

## Описание проекта
Это приложение «Simple To-Do List» — трекер задач, сверстанный в точности по Figma-макету. Основной упор сделан на **переиспользование компонентов**, чистую архитектуру и методологию **TDD (Test-Driven Development)** с полным покрытием логики автотестами (Jest + React Testing Library).

## Дизайн-референс (Figma)
- Источник макета: [Figma Community — Simple To-Do List](https://www.figma.com/community/file/1549999685602720403)
- Макет **не является моей авторской работой**: использован как публичный дизайн-референс для учебной/портфолио-верстки.
- Цель проекта — показать реализацию интерфейса максимально близко к исходному макету (структура, отступы, состояния, светлая/тёмная тема).

## Функционал
- **Сайдбар** с информацией пользователя, навигацией и переключателем темы.
- **Главный экран** со списком задач, полем добавления и счетчиком активных дел.
- **Управление задачами**: создание, чтение, редактирование (inline), удаление (CRUD).
- **Фильтрация**: Все (All), Активные (Active), Завершенные (Completed).
- **Темная/Светлая тема**: полное соответствие макету в обоих режимах, сохранение предпочтения в `localStorage`.
- **Интеграция с локальным API (`json-server`)**: автоматическая загрузка 10 задач при старте с обработкой состояния загрузки и возможных ошибок сети (кнопка `Retry`).
- **Локальное хранилище**: локально созданные задачи сохраняются в браузере (в начале списка) и не пропадают при перезагрузке.

## Архитектура
Проект разбит на изолированные слои в директории `src/`:
- `components/` — разделены на `layout/`, `tasks/` и `ui/`. Компоненты UI (Button, Checkbox, ToggleTheme) глупые и 100% переиспользуемые.
- `hooks/` — `useTasks` и `useTheme` выносят весь "умный" state-management и сайд-эффекты из компонентов. `TasksPage` теперь выступает лишь как сборщик интерфейса без тяжелой бизнес-логики.
- `lib/` — чистая логика (`logic/`), API (`api/`) и обертки для хранилища (`storage/`), покрытые Unit-тестами.

Мы не использовали сложные state-менеджеры вроде Redux, ограничившись встроенными хуками React (`useState`, `useEffect`, `useCallback`, `useMemo`), что идеально подходит под масштаб проекта.

## Скрипты для работы
- `npm run api` — запуск локального API на `http://localhost:3010/todos`
- `npm run dev` — запуск API и Vite dev сервера вместе
- `npm run dev:web` — запуск только Vite dev сервера (без API)
- `npm run test` — запуск всех Unit и Integration тестов (через Jest)
- `npm run lint` — запуск проверки кода через ESLint
- `npm run build` — сборка проекта для продакшена

## Стек технологий
- **Framework:** React (Functional Components, Hooks)
- **Язык:** JavaScript (ES6+)
- **Стилизация:** CSS Modules
- **Сборщик:** Vite
- **Тестирование:** Jest + React Testing Library + jest-dom
- **API:** Fetch API / Promises

## Установка и запуск
1. Склонируйте репозиторий
2. Выполните `npm install`
3. Выполните `npm run dev` для запуска API и приложения
4. Выполните `npm run test` чтобы убедиться, что все TDD-тесты зелёные
